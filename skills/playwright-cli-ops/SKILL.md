# Playwright-cli Ops

## 1. 适用场景

在以下场景触发：

1. 需要通过浏览器执行搜索。
2. 需要打开候选页面并提取正文。
3. 需要把页面内容导出为后续清洗输入。

不要在以下场景触发：

1. 本地知识已经足够，不需要联网。
2. 已经拿到页面正文，只需要做清洗或持久化。

## 2. 职责边界

本 skill 只负责“如何用 Playwright-cli 稳定地拿到网页内容”：

1. 优先使用官方 `playwright-cli` 命令。
2. 如果项目本地已安装 package，可使用 `npx --no-install playwright-cli`。
3. 优先做搜索，再做候选页筛选，再做页面打开与内容导出。
4. 记录标题、URL、站点、页面类型、命中原因。

本 skill 不负责：

1. 把网页内容清洗成最终知识文本。
2. LLM 分块。
3. Milvus 或本地文件持久化。

## 3. 使用规则

1. 始终优先调用官方 `playwright-cli` 命令。
2. 如果命令不存在，明确报错并提示安装 `npm install -g @playwright/cli@latest` 或在项目中本地安装。
3. 搜索时优先带上站点限定词、产品名、版本词和主题词。
4. 打开页面前先做候选页筛选，尽量排除导航页、搜索页、广告页、聚合页。
5. 抓取时保留标题、正文、代码块、表格、FAQ、步骤说明。
6. 失败时直接暴露命令失败，不要静默切换到别的抓取实现。

### 3.1 SPA 页面处理（硬约束）

许多现代文档站（如 coze.com、vercel.app 等）是纯 SPA，页面正文由客户端 JavaScript 渲染。直接抓取 HTML 只会拿到空壳（大量 `<script>` 标签，无实质正文）。必须遵循以下规则：

1. **渲染等待**：打开页面后，必须等待页面核心内容渲染完成再提取。使用 Playwright 的 `waitForSelector` 或 `waitForLoadState('networkidle')` 等待机制，确保 JS 执行完毕。
2. **空壳检测**：如果提取到的正文长度 < 500 字符，且 HTML 中包含大量 `<script>` 标签但缺少 `<article>` / `<main>` / 有意义的 `<p>` 标签，判定为 SPA 空壳，必须重新等待渲染后提取。
3. **重试策略**：首次提取为空壳时，等待 5 秒后重新提取，最多重试 2 次。如果仍然为空壳，在输出中标注 `spa_render_failed: true`，由上游 skill 决定降级处理。
4. **禁止伪造**：如果页面内容无法通过 Playwright 提取（如需要登录、反爬严格），**禁止用 LLM 训练知识自行编写内容**。必须在输出中明确标注抓取失败原因。

## 4. 输出要求

输出至少包含：

1. 本轮执行的 Playwright-cli 命令。
2. 候选页面列表。
3. 最终选中的页面。
4. 导出的原始页面内容或正文文本。
